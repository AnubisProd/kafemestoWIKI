# Настройки SSL (HTTPS) на Vercel по состоянию на март 2026 года. 
>[ !WARNING ]Хорошая новость: в 99% случаев ничего настраивать вручную не нужно — Vercel делает всё автоматически.
>


Вот подробное объяснение **настройки SSL (HTTPS)** на Vercel по состоянию на март 2026 года. Хорошая новость: **в 99% случаев ничего настраивать вручную не нужно** — Vercel делает всё автоматически.

### Как работает SSL на Vercel (автоматически)

- Vercel использует **Let's Encrypt** для генерации бесплатных SSL-сертификатов.
- Сертификаты выдаются **автоматически** для:
  - Всех preview- и production-деплойментов на `*.vercel.app` (wildcard-сертификат)
  - **Всех кастомных доменов**, которые ты добавил в проект и правильно верифицировал через DNS
- HTTPS включается **принудительно**: любой запрос по HTTP → редирект 308 на HTTPS
- Сертификаты **автоматически обновляются** (renewal) — тебе не нужно ничего делать
- Поддерживаются как обычные домены, так и wildcard (*.example.com) — но для wildcard нужно делегировать NS на Vercel

### Пошаговая настройка SSL для твоего домена (kafemestoyalta.ru / .com)

1. **Добавь домен в проект** (если ещё не сделал):
   - Vercel Dashboard → твой проект → **Settings** → **Domains**
   - Введи `kafemestoyalta.ru` (или `www.kafemestoyalta.ru`) → **Add**
   - Добавь оба варианта, если хочешь (apex + www)

2. **Верифицируй домен (DNS-записи)**:
   Vercel покажет точные инструкции (они уникальны для твоего домена).

   | Тип домена          | Что добавить у регистратора (Reg.ru, Namecheap и т.д.) | Пример значения (Vercel покажет точное) |
   |---------------------|---------------------------------------------------------|------------------------------------------|
   | Apex (kafemestoyalta.ru) | A-запись                                               | 76.76.21.21 (или актуальный IP от Vercel) |
   | www (www.kafemestoyalta.ru) | CNAME-запись                                          | cname.vercel-dns.com (или хэш вроде d1d4fc829fe7bc7c.vercel-dns-017.com) |
   | Альтернатива (самый простой) | Делегируй NS на Vercel                                 | ns1.vercel-dns.com + ns2.vercel-dns.com (или больше) |

   - После изменения DNS жди **пропагации** (обычно 5–60 минут, иногда до 2 часов).
   - Vercel автоматически проверяет → статус изменится на **Valid Configuration**.

3. **SSL-сертификат выдаётся автоматически**:
   - Как только DNS верифицирован → Vercel запускает **HTTP-01 challenge** от Let's Encrypt (для обычных доменов) или **DNS-01** (для wildcard).
   - Процесс занимает от 30 секунд до 10–15 минут.
   - В разделе Domains увидишь статус:
     - **Certificate Status**: Active / Valid
     - **HTTPS**: Enabled (с зелёным замком)
   - Если статус **Pending** или **Invalid** > 30 мин → проверь DNS (используй `dig` или whatsmyip.org/tools/dns-propagation-checker).

4. **Проверь работу**:
   - Открой https://kafemestoyalta.ru
   - В браузере → замок → Certificate → Issued by: Let's Encrypt
   - Если видишь ошибку (SSL_ERROR_SYSCALL, handshake failure и т.п.) → типичные причины:
     - DNS ещё не пропагировался полностью
     - Конфликт записей (старые A/CNAME остались)
     - Домен добавлен в другой проект Vercel (удали оттуда)
     - Редко: баг на edge Vercel — подожди 1–2 часа или обратись в support

### Дополнительные опции и нюансы

- **Принудительный HTTPS** — уже включён по умолчанию (308 redirect). Можно отключить в Domains → Edit → HTTPS → но не рекомендуется.
- **Wildcard-домены** (*.kafemestoyalta.ru) — только если NS делегированы на Vercel → сертификаты выдаются индивидуально для каждого субдомена on-the-fly.
- **Кастомный SSL-сертификат** (свой, купленный у CA) — доступно только на **Enterprise-плане**. На Hobby/Pro — только автоматический от Let's Encrypt.
- **Если сертификат не выдаётся** (самая частая проблема):
  - Проверь, что путь `http://kafemestoyalta.ru/.well-known/acme-challenge/...` доступен (Let's Encrypt проверяет именно его).
  - Убедись, что нет Cloudflare proxy или другого CDN перед Vercel (отключи orange cloud в Cloudflare, если используешь).
  - Добавь домен заново → иногда помогает.

### Итог: что тебе нужно сделать прямо сейчас

1. Добавь домен → следуй инструкциям Vercel по DNS.
2. Подожди 10–60 минут.
3. Проверь статус в Domains → должен быть зелёный замок и "Active".
4. Всё! HTTPS работает, сертификат обновляется сам.

Если через час статус всё ещё "Pending" или "Invalid" — пришли скриншот раздела Domains (с DNS-инструкциями и статусом) — помогу разобрать конкретно твою ситуацию.

Удачи с запуском kafemestoyalta.ru на полном HTTPS! 🔒
