# Инструкции.

## 1. Привязка своего домена (kafemestoyalta.ru / .com / .su и т.д.)

### Шаги в Vercel Dashboard:

1. Зайди в свой проект → **Settings** (в боковом меню слева) → **Domains**.
2. Нажми большую кнопку **Add** (или **Add Domain**).
3. Введи домен полностью:
   - Для **apex-домена** (без www): `kafemestoyalta.ru` или `kafemestoyalta.com`
   - Для **поддомена**: например `www.kafemestoyalta.ru`
   - Рекомендую добавить оба варианта (apex + www), а потом настроить редирект.
4. Нажми **Add** → Vercel покажет статус **Invalid Configuration** или **Needs Configuration** и даст точные инструкции по DNS.

> **Что обычно нужно сделать в DNS у регистратора** (Reg.ru, Namecheap, GoDaddy, Cloudflare и т.д.):

| Тип домена       | Рекомендуемый DNS-запись                  | Значение (пример)                                      | Примечание |
|------------------|-------------------------------------------|--------------------------------------------------------|------------|
| Apex (kafemestoyalta.ru) | A-запись                                 | 76.76.21.21 (или актуальный IP от Vercel — всегда берётся из их инструкции) | Один или несколько A-записей, если Vercel просит |
| www (www.kafemestoyalta.ru) | CNAME-запись                             | cname.vercel-dns.com (или уникальный хэш вроде d1d4fc829fe7bc7c.vercel-dns-017.com — Vercel выдаст точный) | Точный CNAME Vercel даёт после добавления домена |
| Альтернатива (проще, но дольше) | Vercel Nameservers                       | Скопировать 2–4 NS от Vercel (ns1.vercel-dns.com и т.д.) и вставить у регистратора | Всё остальное (MX, TXT и т.д.) настраивается уже в Vercel DNS |

- После изменения DNS жди **пропагации** (от 5 минут до 48 часов, чаще 15–60 мин).
- Vercel сам проверяет и обновляет статус → когда увидишь **Valid Configuration** + зелёный замок → HTTPS включён автоматически (бесплатный Let's Encrypt).
- Если домен уже где-то использовался (даже в другом аккаунте Vercel) → может потребоваться TXT-запись для верификации владения.

>[NOTE]**Совет**: если у тебя Reg.ru или российский регистратор — часто проще сразу делегировать на Vercel Nameservers, чтобы не мучаться с A/CNAME.

После успешной привязки:
- Production-деплой → будет открываться по твоему домену
- Старый `.vercel.app` остаётся работать (можно удалить, если не нужен)

### 2. Включение Automatic Redeploys на каждый push

Это уже включено **по умолчанию** для всех проектов, подключённых к GitHub.

- Каждый `git push` в production-ветку (обычно **main** или **master**) → Vercel автоматически запускает новый билд и деплой.
- Проверить / подтвердить:
  1. В проекте → **Settings** → **Git** (или **Git Integration**).
  2. Убедись, что:
     - Repository подключён правильно
     - **Production Branch** = main (или твоя основная ветка)
     - Нет кастомного **Ignored Build Step** (если там стоит что-то вроде "Don't build" — удали)
  3. Если авто-деплои не срабатывают:
     - Переподключи репозиторий (Reconnect GitHub)
     - Проверь webhook в GitHub → Repository → Settings → Webhooks → должен быть один от Vercel (если нет — проблема в интеграции, удали и добавь заново Vercel app в GitHub)

В 99% случаев ничего включать не нужно — просто пушь в main, и через 20–90 секунд увидишь новый деплой в **Deployments**.

### 3. Preview Deployments — каждый PR получает свою тестовую ссылку

Тоже работает **автоматически по умолчанию** (Vercel for GitHub).

Что происходит:
- Создаёшь Pull Request (из любой ветки в main) → Vercel запускает билд
- После успешного билда → в PR появляется комментарий от **Vercel** с ссылкой вида:
  ```
  https://kafemestoyalta-git-feature-branch-yurydevru.vercel.app
  ```
  (или похожей — зависит от имени проекта и ветки)
- Каждый новый push в эту ветку / PR → обновляет тот же preview (или создаёт новый, если сильно изменилось)

**Как убедиться, что всё включено**:
1. **Settings** → **Git** → посмотри, что **Preview Deployments** не отключены (по умолчанию включены).
2. Если хочешь отключить previews для определённых веток → используй **Ignored Build Step** с bash-командой (редко нужно).
3. Лимитов на количество preview-деплойментов практически нет (сотни в месяц бесплатно на Hobby-плане).

**Полезно**:
- В PR можно кликнуть на preview-ссылку → протестировать изменения без риска для продакшена
- После merge PR в main → preview исчезает, изменения идут в production

Если сделаешь эти три шага — твой сайт будет полностью профессионально настроен:
- кастомный домен с HTTPS
- авто-деплой на каждый push в main
- безопасные тестовые окружения для каждого PR

Если при настройке домена Vercel выдаст нестандартные инструкции (редко, но бывает) — пришли скрин или текст ошибки, помогу разобрать. Удачи с запуском кафе-сайта! ☕
